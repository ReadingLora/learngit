# 第 2 周复盘（2026-05-10）

## HTML、CSS、JS 分别负责什么（本笔记合并说明，不单独建 web-basic-notes）

下面是我现在对三者的理解，之后会随着练习再改表述。

- **HTML**：页面的结构。其实就像我排布游戏剧本的场景布局图一样，有了场景布局图，才能进一步和美术部门进行沟通。
- **CSS**：页面的表现（界面效果）。美术部门在场景布局图的基础上，进一步画出详细的设计图、氛围图，这就是CSS。
- **JavaScript（JS）**：页面的互动设计。对应在游戏里，即是“实现”，最终美术部门具体在游戏中搭建出整个场景，且场景中的NPC、物件等可与玩家产生互动。

一句话：**HTML 搭骨架，CSS 做外观与布局，JS 管交互与变化。**

## 十道前端基础题对练（2026-05-10）——整体表现

对练未逐题存档，仅作整体点评。外置 CSS、复用与维护成本，以及 API Key 与额度风险，**工程直觉清楚**；盒模型内外、`push`/`map` 方向也对。结合 `style.css` 讨论 Flex 后能分主轴与交叉轴，并纠正「`space-between` 不是双居中」，**对照实作样式反推**比死记定义更有效。

薄弱处：**概念边界**——语义化曾误举 `<span>`、`var` 与「事件」曾混谈，需把无语义与语义标签、声明与监听**分栏记**。`getElementById` 上手快，`querySelector` 已能联想 CSS 选择器；仓库全 `var` 曾让 `const`/`let` 滞后，澄清后逐步改代码即可。
