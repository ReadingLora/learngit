## 三个核心概念（本周真正用上的）

### 1. 数据结构 + 单一数据源

```text
Book = { id, title, author, blocks: [] }
Block = { id, type, content, createdAt }
```

所有页面只认 **`mock-books.js`（及 localStorage 里的副本）**，避免「首页一套、阅读页一套」。

### 2. 本地存储

- 存：**对象 → JSON 字符串 → `localStorage`**
- 取：**字符串 → `parse` → 对象**
- 按 **`bookId`** 分抽屉；刷新不丢（原型阶段验证手段，桌面版以后换文件/库）。

### 3. 状态 → 布局（`f` + class）

- **`body.*-collapsed`**：哪一栏关着。
- **`layout()`**：根据几栏打开，算可用宽度、显示哪条 gutter、写各栏 `flex` 像素。
- **`f`**：三栏宽度比例；切换组合时必须保证「当前打开的几栏，比例之和有意义」，否则会出现某一栏只剩 `MIN_R` / `MIN_N` 的「假展开」。

## `reader-layout` 现在能做什么 / 不能做什么

**能做的（网页原型）**

- 书架进书 → 三栏阅读页；中栏模拟 AI 回答 → **加入笔记** → 右栏折叠列表。
- 笔记 **按书保存**，刷新仍在。
- **导出**当前书全部笔记为 Markdown（手动复制）。
- 顶栏 **三栏独立收起/展开**、拖拽调宽（含 AI 收起时阅读↔笔记）。
- 无效书籍参数回首页。

**还不能做的（刻意不做或下周以后）**

- 真实 AI 对话、API Key 进前端（安全上也不做）。
- 右栏笔记 **展开后编辑并写回**（计划里 5/13 项，仍待做）。
- 按类型「新增摘录块 / 问题块」等独立按钮（5/12 计划项）。
- 一键写入 Obsidian  vault 文件（当前仅剪贴板/弹窗复制）。
- Electron 桌面壳、微信读书 WebView。
