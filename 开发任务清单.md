# 开发任务清单（Electron + React｜阅读器 V1.0 MVP）

目标（对齐学习计划 V2.0）：完成「阅读 → 摘录 → AI 提问 → 块笔记整理 → 导出 Markdown → 放入 Obsidian 归档」闭环。  
不做：PDF 导出、完整对话历史系统、自动抓取微信读书正文/划线、SQLite 深度使用（后续再说）。

---

## 顶部对照：每个任务对应第几周（你不用来回翻两份文件）
- **Week 1**：产品说明 → 任务清单（你现在正在做）
- **Week 2**：静态界面草图（HTML/CSS 三栏布局、书籍卡片）
- **Week 3**：数据结构 + localStorage 保存 + **导出 Markdown（网页原型）**
- **Week 4**：用函数/模块重构网页原型（为 React 做准备）
- **Week 5**：React + Vite 重构（组件化、state、本地保存恢复）
- **Week 6**：块编辑器（折叠 + 拖拽排序 + 块类型）
- **Week 7**：AI API 接入（Node 代理服务，前端不碰 Key；AI 回答自动保存为块）
- **Week 8**：Electron 桌面封装 + 微信读书嵌入（或并排方案）+ 安全设置
- **Week 9**：写作工作台（写作草稿区）+ Markdown 导出模板优化 + 整书导出
- **Week 10**：打包、测试、文档、安全审查、路线图
- **Week 11-14**：真实使用、Bug 修复、技术深化、总复盘（按计划执行即可）

---

## 0) 基础约定（全局）
- 渲染进程（React）：页面与组件
- 主进程（Electron main）：窗口/WebView、文件读写（导出）、安全设置、与渲染进程通信
- preload：用 `contextBridge` 暴露 `window.api.*`（React 不直接用 Node 能力）
- 存储策略（对齐计划）：**第一轮用 localStorage 跑通；Week 6 只评估 Dexie/IndexedDB，暂不强迁移；SQLite 属于 V1.0 之后再考虑**

## 7) AI API 接入（Node 代理服务 + 自动保存 AI 回答块）（Week 7）
> 本周重点是**安全**：Key 永远不进前端代码、不进 Git、不发到聊天框。

### 7.1 DeepSeek/Gemini API 独立测试脚本（Node）
- 周次：Week 7
- 难度：中等
- 技术：Node 脚本 + 从系统环境变量读取 Key

### 7.2 本地 API 代理服务（Express）
- 周次：Week 7
- 难度：困难
- 技术：Express 起本地服务 `/api/ask`，在服务端读取 Key 并调用 AI
- 接口（概念上）：
  - POST `/api/ask`：{ excerpt, question, skill? } -> { answer }

### 7.3 React 阅读器接入代理服务：提问 -> 回答 -> 自动保存为 AI 回答块
- 周次：Week 7
- 难度：中等
- 技术：前端 fetch 调用本地服务 + 将结果写入 blocks

### 7.4 默认 Prompt（Week 7 只做 1 个硬编码版本）
- 周次：Week 7（只做默认 1 个）；Week 9（扩展成 3-5 个 + 下拉切换）
- 难度：简单 → 中等（Week 9 扩展时）
- 技术：prompt 字符串拼接（Week 7）；配置化模板 + 下拉 UI（Week 9）
- Week 7 交付：
  - 只内置 1 个默认 prompt（硬编码），例如：「把这段摘录转为卡片笔记」
  - 不做 Skill 下拉切换 UI
- Week 9 扩展：
  - 增加 3-5 个模板（摘要/卡片笔记/公众号大纲等）
  - 增加下拉选择，并把选择结果传给 AI 请求

### 7.5（V2 再考虑）完整对话历史系统
- 周次：V2 再考虑
- 难度：中等
- 技术：对话线程/消息存储与加载

> Week 8 的目标是“桌面雏形跑通”，嵌入失败也允许切换并排方案。

### 8.1 最小 Electron 窗口加载 React
- 周次：Week 8
- 难度：中等

### 8.2 微信读书嵌入尝试（webview）或并排方案
- 周次：Week 8
- 难度：中等
- 技术：Electron webview 嵌入远程网页（如受限就转“外部浏览器并排”）
- 说明：第一版允许手动复制摘录，不做自动抓取。
- 失败判定标准：如果 webview 加载后无法正常登录微信读书、或页面持续白屏（多次重启/刷新仍无效），即判定嵌入方案本阶段不可行，切换为「外部浏览器打开微信读书 + 阅读器并排」方案，并在复盘里记录失败原因。

### 8.3 Electron 安全设置
- 周次：Week 8
- 难度：困难
- 技术：禁用远程页面 Node 权限、最小化权限、按官方安全清单检查
