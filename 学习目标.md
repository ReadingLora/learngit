## 今日学到的核心概念

### 1. 数据结构设计
- Book = { id, title, author, blocks: [] }
- Block = { id, type, content, createdAt }
- 所有书的信息集中在一个地方定义，其他文件从这里读取——「单一数据源」原则

### 2. JSON.stringify 和 JSON.parse
- localStorage 只能存文字，不能存对象
- 存入时用 JSON.stringify 把对象转成文字（拍照变纸条）
- 读取时用 JSON.parse 把文字转回对象（纸条还原成实物）
- 这对搭档是 localStorage 存取数据的标配

### 3. URL 参数（?book=jzs）
- 问号后面附带的额外信息，告诉页面当前打开的是哪本书
- script.js 中 currentBookId() 函数读取这个参数
- 实现「笔记隔离」：每本书的笔记存在各自的格子里，互不干扰

### 4. 动态生成 HTML
- document.createElement() 凭空创建元素
- appendChild() 把元素塞进页面
- 之前的 HTML 标签是手写的（静态），现在由 JS 根据数据自动生成（动态）
- 首页书籍卡片、阅读页笔记卡片都是动态生成的

### 5. CSS 变量（--开头）
- 定义一次，到处引用：--shelf-cover-h: 35vh
- 改一个数字，所有相关尺寸自动联动
- 和编程里的变量是同一个思路

### 6. 脚本加载顺序
- mock-books.js 必须在 script.js / shelf-home.js 前面加载
- 否则后面的脚本找不到 READER_MOCK_BOOKS 数据
- 顺序错了页面不会报错但功能会失效

### 7. 通过 class 切换控制样式状态
- JS 负责「什么时候加/去掉 class」
- CSS 负责「这个 class 存在时长什么样」
- 例：加入笔记后，父容器加上 ai-mock-answer--synced，文本框自动变绿

## 今日学到的核心概念

- if (条件) { 条件成立时执行 }
- if...else：条件成立走前面，不成立走后面
- 条件可以用 &&（与）和 ||（或）组合多个判断

### 2. && 和 || 的本质

- &&（与）：false 偏向，找到第一个假值就返回它，找不到就返回最后一个值
- ||（或）：true 偏向，找到第一个真值就返回它，找不到就返回最后一个值
- 这叫「短路求值」——不需要的条件根本不会去算
- 2 && 3 返回 3（两个都是真，返回最后一个）
- 2 || 3 返回 2（第一个就是真，直接返回）

### 3. 数字转布尔值规则

- 0 和 NaN → false
- 其他所有数字 → true
- 空字符串 "" → false，非空字符串 → true
- null → false，undefined → false
- 对象 → 永远是 true

### 4. return 的意思

「到此为止，后面的代码不执行了」
函数里遇到 return 就立刻离开这个函数

### 5. try...catch 错误防护网

- try 里面的代码如果出错，不让页面崩掉，跳到 catch 里处理
- script.js 里的 catch 处理方式：出错就返回 null，当作没有数据

### 6. typeof 和 Array.isArray()

- typeof 检查变量的类型（是 undefined？string？number？）
- Array.isArray() 检查一个东西是不是数组
- 两者经常用来做「数据体检」——确认数据存在且格式正确

### 7. .length 属性

- 数组的 .length 返回里面有几个元素
- 字符串的 .length 返回有几个字符
- .length 为 0 转布尔值是 false，用 !xxx.length 判断「是否为空」

## 三、JavaScript 核心概念整理

### for 循环的三个要素
```
for (初始化; 条件; 更新) { 循环体 }
```
- **初始化（initialization）**：var i = 0 → 从哪里开始
- **条件（condition）**：i < array.length → 什么时候停
- **更新（update）**：i++ → 每轮结束后换下一个

### .length 属性
- 数组自带属性，返回数组里有几个元素
- READER_MOCK_BOOKS.length = 3（3本书）
- KNOWN_BOOK_IDS.length = 3（3个id）
- notes.length = 当前书有几条笔记

### if 和 for 是最常用的两种语句
- if 解决「判断」：数据存在吗？格式对不对？该走哪条路？
- for 解决「重复」：有一组数据，每个都要做同样的处理
- while 是不知道重复几次的 for，switch 是同一个变量比较多次的 if

### JavaScript 的 7 种数据类型

**基本类型（5种）**：
| 类型 | 说明 | 代码实例 |
|------|------|---------|
| String | 文字，用引号包裹 | "reader-layout-store-v2" |
| Number | 数字 | 4000、0.4 |
| Boolean | 真/假 | true、false |
| Undefined | 声明了但没赋值 | var x; |
| Null | 主动表示「空」 | var drag = null |

**复合类型（2种）**：
| 类型 | 说明 | 代码实例 |
|------|------|---------|
| Object | { key: value } 键值对 | BOOK_META、每条笔记 |
| Array | [ ] 有序列表 | KNOWN_BOOK_IDS、notes |

### 对象的三种分类（按来源）
- **内建对象**：JS 语言自带 → Date.now()、JSON.parse()、Math
- **宿主对象**：运行环境提供 → document、localStorage、window
- **自定义对象**：自己写的 → BOOK_META、createBlock() 返回的笔记对象
