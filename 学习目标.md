## 今日学到的核心概念

### 1. 数据结构设计
- Book = { id, title, author, blocks: [] }
- Block = { id, type, content, createdAt }
- 所有书的信息集中在一个地方定义，其他文件从这里读取——「单一数据源」原则

### 2. JSON.stringify 和 JSON.parse
- localStorage 只能存文字，不能存对象
- 存入时用 JSON.stringify 把对象转成文字（拍照变纸条）
- 读取时用 JSON.parse 把文字转回对象（纸条还原成实物）
- 这对搭档是 localStorage 存取数据的标配

### 3. URL 参数（?book=jzs）
- 问号后面附带的额外信息，告诉页面当前打开的是哪本书
- script.js 中 currentBookId() 函数读取这个参数
- 实现「笔记隔离」：每本书的笔记存在各自的格子里，互不干扰

### 4. 动态生成 HTML
- document.createElement() 凭空创建元素
- appendChild() 把元素塞进页面
- 之前的 HTML 标签是手写的（静态），现在由 JS 根据数据自动生成（动态）
- 首页书籍卡片、阅读页笔记卡片都是动态生成的

### 5. CSS 变量（--开头）
- 定义一次，到处引用：--shelf-cover-h: 35vh
- 改一个数字，所有相关尺寸自动联动
- 和编程里的变量是同一个思路

### 6. 脚本加载顺序
- mock-books.js 必须在 script.js / shelf-home.js 前面加载
- 否则后面的脚本找不到 READER_MOCK_BOOKS 数据
- 顺序错了页面不会报错但功能会失效

### 7. 通过 class 切换控制样式状态
- JS 负责「什么时候加/去掉 class」
- CSS 负责「这个 class 存在时长什么样」
- 例：加入笔记后，父容器加上 ai-mock-answer--synced，文本框自动变绿
